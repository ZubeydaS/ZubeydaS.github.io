Welcome to My Creative Coding and Generative AI Class Projects

This class was an exploration of combining creativity with machine learning and generative algorithms. I worked on a variety of projects that involved building interactive and generative systems using code and AI technologies. Below are three projects that showcase my journey and the skills I developed throughout the course.

Course Description:
Generative AI tools such as ChatGPT and GitHub CoPilot are fundamentally reshaping programming practices and workflows, raising questions about the future of code and so-called “prompt engineering,” or writing for the machine. This class will situate this moment of potential transformation in the history of literate programming and “natural language” coding using Inform 7, as well as current tools such as ml5.js, an accessible machine learning library. Students will engage this history and future of computational creativity through writing and re-writing code, both with and without generative AI interventions, for conversational bots, interactive fiction, and experimental games.

The course is divided into three modules: Writing as Code; Code as Text; and Creative Code. Throughout, we will work together on weekly shorter creative coding exercises and build towards three more complicated projects: a parser-based interactive fiction; a text generator; and a portfolio of interactive art.

All of the projects I created incorporated the use of AI, in line with the course's focus on understanding AI tools. However, the main objective was to explore how creative spins can be added through the combination of AI and human imagination. We also made a conscious effort to distinguish between AI-generated work and our own, developing a better sense of AI’s capabilities and forming opinions on the ethics of AI usage through readings, lectures, and practice.

Assignment Instructions:

Creative Code with ML5
This week, we’ll be working from our experiments with Teachable Machine and ml5 to build an original web implementation of a responsive interface. This should be a creative project and doesn’t need to have any “purpose” beyond an aesthetic response to webcam input.

In class, we’ll work through these examples and learn from their approaches:

  • Kinetic Type and FaceMesh
  • Falling Coins
  • Interactive Fridge Magnets

I recommend the ml5.js Coding Train Videos for additional references on the current capabilities of the library.

Submit either an OpenProcessing / P5.js link or a GitHub Pages deployment of your experiment.

Workflow: Using Teachable Machine and Ml5, I created a web interface that would use input from a users camera to create unique affects based on the Teachable Machine model. This assignment used both AI in creation of the affects. The model was created with Teachable Machine. The model is trained to recognize three hand symbols and/or poses, and show their accuracy with varying effcts. The choice of the training model was based on jokes my friends and I tend to make. The Teachable Machine model was also very sensative and difficult to train since the model was only based on myself, without varying subjects.

Project 2: P5.js: Interactive Cat Room Game

Click to learn more and view the full project!

View the Interactive Cat Game

Assignment Instructions:

Exercise: P5 Game
Using a familiar genre as a starting point, build out a complete, one room / one level, game in P5.js. I’ve built out an extensive example of this week’s approach in AI Admin. Notice how this work becomes a metagame considering the potential role of generative AI tools in shaping interactive experiences for education and training. It’s also built on very simple conditionals. The game’s primary logic can all be described under the framework of “if / else” - players are given simple scenarios and a choice of answers, and each answer in turn changes the player’s reputation on leading AI policy for their department.

Building for Interaction
Here’s an overview of the steps / prompts involved - however, you’ll need to adjust these to your goals. Decide when to program directly, and when using generative AI tools to build or debug will be most useful.

  • Set Up the Canvas
  • Load a Background Image
  • Display Mouse Coordinates
  • Define Mouseover Zones Using Coordinates
  • Make Zones Clickable and Display Messages
  • Build Interactions
  • End the Game After All Interactions

Submit the link to your live, GitHub Pages or Open Processing version of the game on Moodle.

Workflow:
This game was created because I found a cute image online of a cat in a cozy room, and I wanted to build a game that captured that same comforting vibe. The image inspired me to create a simple, interactive game based on the cozy atmosphere of the cat in bed. I thought it would be fun to design a game that uses familiar "iSpy" elements, as I enjoyed those books as a kid. The objective is for the player to find hidden items within the room while interacting with the environment in a way that feels natural and cozy.

Building on the concepts from my professor's demo and using the P5.js library, I wanted to make the game visually appealing with cute elements and simple mechanics. The user can click on various objects within the cat’s room, and each interaction reveals something new, whether it's a playful response or an item the player needs to find.

I focused on making the game easy to play with simple instructions, while still offering enough engagement to capture the fun elements of a point-and-click game. Additionally, I designed the game with aesthetics in mind, ensuring the visuals were cute and cozy, matching the theme of the cat in bed.

Sources: P5.js documentation and library, OpenAI, and my professor's example Ai Admin

Assignment Instructions:

Bluesky Bot
In this assignment, you’ll use BlueBotsDoneQuick and Tracery to design an original generative grammar for both text and images, tailored to post on a Bluesky account. Use the sample code and deployed bot sample as a guide, and play with abstractions and combinatorial poetics.

As you work, try using generative AI tools to:

  • Develop the structure of your SVG. Iterate through this process without mentioning Tracery to focus on creating a representation of the abstraction, landscape, etc you want to generate.
  • Generate possibilities for each element of your grammar. Generate as JSON, and use prompts like web-safe color names, values within a range, etc, to speed up your Tracery grammar crafting process.
  • Develop your initial Tracery grammar. Make sure to check that the grammar ties the output to the origin so that it will work in the Bluesky pot structure.

Make sure to select “display source” when deploying your bot, and set your bot to post every twelve hours. Submit the link to your bot’s hosted source code as per the class example to the Moodle for credit for this exercise.

Workflow: In this assignment, I used BlueBotsDoneQuick and Tracery, as well as a class demo from the professor to design a generative grammar for both text and images, tailored to post on a Bluesky account. The bot combines generative AI tools with creative grammar structures to generate unique content. I iterated through various stages, developing SVG representations and experimenting with color ranges and scenery to display changing text and imagery of both a cat and poetry akin to day to day concerns of anxiety and imposter syndrome.

Sources: Tracery, class demo resource, BlueBotsDoneQuick, OpenAI
